# 音声会話（往復ループ）

Claude の返答を読み上げ、ユーザーの声を聞き取って、会話をループで続ける。

## 前提の確認（最初に1回）

`claude-talk` MCP の 2 ツールが使える必要がある:
- `speak` … テキストを読み上げる（完了まで待つ）
- `listen` … マイクの発話を認識してテキストで返す

これらが見当たらない、または最初の呼び出しで「音声アプリが起動していません」というエラー（`isError`）が返る場合は、**ループに入らず**にこう伝えて止める:
> 音声アプリ（claude-talk-mcp）が起動していないか、MCP が未登録の可能性があります。アプリをメニューバーに常駐させてから、もう一度 /voice を実行してください。

## 会話ループ

1. **読み上げる** — 今の返答を `speak` で読み上げる。読み上げ用に整形してから渡す（下記「話し言葉の作法」）。返答の最後は**短い合図**（「どうぞ」等）で締め、ユーザーがいつ話せばよいか分かるようにする。合図は一言で十分（長い促しは gap を増やすだけ）。
2. **聞き取る** — 合図のあと、**余計な発話を挟まず即** `listen` を呼ぶ（`locale` 既定 `ja-JP`、`timeout` 既定 30 秒）。アプリは読み上げ完了直後からマイクを先回りで回している（プリロール）ので、合図の直後にユーザーが話し始めても頭から録れる。`listen` 発行までの間に話されても取りこぼさない。
3. **無音・空結果のとき** — `listen` が空文字を返したら、「聞こえませんでした。もう一度お願いします」と短く `speak` して、もう一度 `listen`。これを最大 2 回まで。それでも空なら「音声を切ります。続けるときは /voice と言ってください」と読み上げてループを終える。
4. **応答して繰り返す** — 認識テキストをユーザーの次の発言として扱い、返答を作り、1 に戻る。
5. **終了条件** — ユーザーが「終わり」「おわり」「ストップ」「やめて」「バイバイ」「またね」等の終了語を言ったら、短い別れの言葉を `speak` してループを終える。`speak`/`listen` がアプリ側エラーを返したときも、その旨を伝えて終える。

最初のターンは、短い挨拶を `speak` してから `listen` で始める（例:「はい、聞いています。どうぞ」）。

## 話し言葉の作法（`speak` に渡すテキスト）

耳で聞いて分かる文章にする。読み上げ前に必ず整形する:

- 短い文で、1 ターン 1 論点。長い説明は要点だけに絞る。
- Markdown・コードブロック・URL・箇条書き記号・絵文字は**読み上げない**。必要なら言葉に言い換える（「ハイフン」「スラッシュ」等）。
- 記号や英略語は読みやすく（例: `~/.claude` → 「ホーム直下のドットクロード」）。
- コードやコマンドそのものは音声に向かない。「詳細は画面に出しますね」と言い、本文はテキストでも表示する。

## 画面表示との併用

音声はあくまで会話のチャンネル。認識したユーザーの発話と自分の返答は、通常どおりテキストでも簡潔に出しておくと、聞き逃しても追える。長い成果物（コード・表・手順）は読み上げず、画面に出して口頭では要約する。

## 補足

- `listen` の確定は「無音約 1.2 秒」または `timeout` 到達（アプリ側実装）。ユーザーが話し終えたら自然に確定する。
- **プリロール**：`speak` 完了直後からアプリがマイクを回し始め、次の `listen` がその音声を引き継ぐ。だから「合図 → 即 listen」で頭が切れない。listen が来ないと約 6 秒で自動停止。読み上げ中はマイクを止める（エコー防止）。
- 明示的に速度や声を変えたい要望があれば `speak` の `rate` / `voice` を使う（未指定はシステム既定）。
